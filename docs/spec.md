# 行動整理アプリ 仕様書

## 1. 概要

ユーザーの「やること」「やりたいこと」「習慣」を整理するアプリ

一般的なToDo管理ではなく、ユーザーの行動を以下の観点で分類し、日々の行動を見える化する

- Routine
- Task
- Hobby

## 2. 目的

ユーザーが自分の行動を以下のように整理できる状態を作る

- 継続したいことを管理する
- やるべきことを管理する
- やりたいことや興味を管理する
- AIによって分類や整理を補助する

## 3. 基本概念

### 3.1 Routine

習慣化したいこと、繰り返し行うこと

例

- 英語学習
- 筋トレ
- 読書
- 散歩

### 3.2 Task

やるべきこと、対応が必要なこと

例

- 予約する
- 支払いをする
- 部屋を片付ける
- 書類を提出する

### 3.3 Hobby

興味があること、好きでやりたいこと

例

- ギター
- 映画
- 哲学
- 個人開発

## 4. 画面構成

### 4.1 メイン画面

画面上部にカテゴリタブを表示する

- Routine
- Task
- Hobby

選択中のタブに応じて、該当カテゴリのアイテム一覧を表示する

### 4.2 アイテム一覧

各カテゴリ内にアイテムをリスト表示する

表示項目

- アイテム名
- 右向き矢印
- 必要に応じて補足情報

### 4.3 AIボタン

画面右上にAIボタンを表示する

AIボタンを押すと、現在のアイテム整理に対する提案を表示する

AIの役割

- アイテムの分類提案
- 重複アイテムの整理
- Routine化できそうなTaskの提案
- HobbyからChallenge化できそうな項目の提案
- 次にやるべき行動の提案

## 5. 基本操作

### 5.1 アイテム追加

ユーザーは任意のカテゴリにアイテムを追加できる

入力項目

- アイテム名
- カテゴリ
- メモ

### 5.2 アイテム編集

ユーザーは登録済みアイテムを編集できる

編集可能項目

- アイテム名
- カテゴリ
- メモ

### 5.3 アイテム削除

ユーザーは登録済みアイテムを削除できる

削除前に確認ダイアログを表示する

### 5.4 カテゴリ変更

ユーザーはアイテムのカテゴリを変更できる

変更方法

- アイテム編集画面から変更
- アイテム詳細画面から変更

## 6. アイテム詳細画面

アイテムをタップすると詳細画面に遷移する

表示項目

- アイテム名
- カテゴリ
- メモ
- 作成日時
- 更新日時

操作

- 編集
- 削除
- カテゴリ変更

## 7. AI整理機能

### 7.1 分類提案

AIはアイテム名とメモをもとに、適切なカテゴリを提案する

例

入力

- 英語学習

提案

- Routine

理由

- 継続的に取り組む内容のため

### 7.2 整理提案

AIは一覧内のアイテムを分析し、整理案を返す

例

- 「読書」はRoutineまたはHobbyのどちらにも該当する可能性があります
- 「英語学習」はRoutineとして毎日15分に分解できます
- 「部屋掃除」はTaskとして期限を設定すると管理しやすいです

### 7.3 提案の反映

AIの提案に対して、ユーザーは以下を選択できる

- 反映する
- 一部反映する
- 無視する

## 8. データモデル

### 8.1 Item

```typescript
type Item = {
  id: string
  title: string
  category: ItemCategory
  memo?: string
  createdAt: string
  updatedAt: string
}
```

### 8.2 ItemCategory

```typescript
type ItemCategory = 'routine' | 'task' | 'hobby'
```

## 9. 画面一覧

| 画面 | 役割 |
|------|------|
| メイン画面 | カテゴリ別にアイテム一覧を表示 |
| アイテム追加画面 | 新規アイテムを追加 |
| アイテム詳細画面 | アイテム詳細を表示 |
| アイテム編集画面 | アイテム内容を編集 |
| AI提案画面 | AIによる整理提案を表示 |

## 10. MVPスコープ

最初に実装する機能

- カテゴリタブ表示
- アイテム一覧表示
- アイテム追加
- アイテム編集
- アイテム削除
- カテゴリ変更
- ローカル保存
- AI提案の表示

## 11. MVP対象外

初期実装では対象外

- ログイン
- クラウド同期
- 通知
- カレンダー連携
- 習慣達成率
- 詳細な統計
- 複数デバイス同期

## 13. 実装方針

まずはシンプルなローカルアプリとして実装する

優先順位

1. Itemモデル作成
2. カテゴリタブ作成
3. カテゴリ別一覧表示
4. アイテム追加
5. アイテム編集
6. アイテム削除
7. AI提案画面
8. ローカル保存

## 14. 完了条件

MVPの完了条件

- ユーザーがアイテムを追加できる
- カテゴリ別にアイテムを確認できる
- アイテムを編集できる
- アイテムを削除できる
- AI提案画面を開ける
- アプリ再起動後もデータが残る
